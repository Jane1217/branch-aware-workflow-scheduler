üìñ User Guide

üéØ What is this platform?

This platform processes large medical images (Whole Slide Images, WSI) to detect and segment cells. You can create workflows with multiple processing jobs that run automatically.

üìù How to create a workflow:

  1. Enter a workflow name - Give it a descriptive name
  2. Add jobs - Click "Add Job" for each image you want to process
  3. Configure each job:
    • Job Type: Choose what to do with the image
      • Cell Segmentation - Detects and segments all cells in the image
      • Tissue Mask - Creates a binary mask showing tissue regions
    • Select Image: Choose from available test images
    • Branch: Jobs in the same branch run one after another. Use different branch names for parallel processing.
    • Depends On: Optional. List job IDs that must complete first (e.g., "job-1,job-2")
  4. Submit - Click "Submit Workflow" to start processing

üîÑ Understanding Branches:

Same Branch = Serial Execution: If two jobs use "branch-1", they run one after another.

Different Branches = Parallel Execution: Jobs in "branch-1" and "branch-2" can run at the same time.

Example: If you have 3 jobs in branch-1 and 2 jobs in branch-2, the system will run 2 jobs in parallel (one from each branch) until all are done.

üë• Multi-Tenant System:

Each user (identified by User ID) can only see and manage their own workflows. At most 3 users can have running jobs at the same time. The 4th user will wait in queue.

üìä Monitoring Progress:

  • Workflows auto-refresh every 2 seconds
  • Connect WebSocket for real-time updates
  • Progress bars show completion percentage
  • Click "View Jobs" to see individual task status
